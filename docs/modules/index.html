fetch

  • fetch(identifier: string): CipherDriver | HashingDriver
    • Defined in index.ts:30

    Obtain an instance of the requested Driver.

    remarks

    Acceptable identifiers are as follows:

    • Any entry from crypto#getCiphers()
    • Any entry from crypto#getHashes()

    If using in TypeScript, make sure to cast the respective driver appropriately to maintain correct typings.

    throws

    Error (sec:violation:id_disabled) - If the requested identifier is disabled or unsafe to use.

    throws

    Error (sec:violation:id_missing) - If the requested identifier is missing or unavailable.

    readonly
    deprecated

    This function is now deprecated and will be removed in 3.2.0, fetch() will be replaced with getCipher() getHasher() and getFeature effective immediately.

    Parameters

    • identifier: string

      The requested driver implementation from the list of acceptable identifiers.

    Returns CipherDriver | HashingDriver

    The pre-configured driver implementation of the specified identifier.
